## 5. 发现的问题与缺陷（按严重度）

### 🔴 HIGH

**H1 — 安全路由单位缺陷（安全关键）**
- 位置：[safety/router.py:31-34](src/hermes_cgm_agent/services/safety/router.py#L31)
- 现象：`point.value < RED_ZONE_LOW_MGDL(54)` / `> 300` 直接比较，但 `point.value` 是**原始单位**。分析层 [metrics.py:34](src/hermes_cgm_agent/services/analytics/metrics.py#L34) 用的是 `value_mg_dl`（已换算），两者不一致。
- 影响：若用户导入/同步的是 **mmol/L** 数据（importer 支持 `GlucoseUnit.MMOL_L`），所有读数（~4–20）都 `< 54` → 红区**全面误判**，叙事被错误阻断；同时高值（mmol 永远 < 300）→ 高红区**永不触发**，真正的高血糖漏报。
- 当前为何没炸：CSV 样例与 Dexcom 均为 mg/dL，掩盖了缺陷。
- 修复：router 改用 `point.value_mg_dl`（与 analytics 对齐），并补单元测试覆盖 mmol/L 输入。

### 🟠 MEDIUM

**M1 — 黄区（YELLOW）完全缺失**
- 位置：router.py 仅 `red_zone` / `clear` 两态。
- 蓝图（PRD §2.3.1）要求三区：黄区 `54–70 或 180–300` → 正常叙事 + 醒目提醒。
- 影响：偏低/偏高但未到红线的情况没有结构化提醒；SOUL 人格里有"偏低/偏高"软话术，但无代码层黄区路由。

**M2 — 分层推送 / 静默即认可 未落地（PRD P3）**
- 位置：全仓无 scheduler/cron/silence 逻辑；executor.py:755 注释声明交给 Hermes gateway/cron，但未接线。
- 影响：蓝图核心交互范式"日 30 字→周模式→月报告、静默即认可"无法运行；`reports.generate` 只能手动触发。
- 备注：PRD 自身将其列为 P3、可后置，属"已知欠账"。

**M3 — L2→USER.md 同步、Live Hermes provider 安装为 spike**
- 证据：dev-status `prototype_limit: L2->USER.md sync and live Hermes provider install are spikes`。
- 影响：L2 语义档案与 Hermes USER.md 的双向同步未生产化；provider 安装链路未经端到端固化。

**M4 — Dexcom API 路径可审计性缺口**
- 现象：CSV 导入会落 `raw_cgm_records`+原始 JSON；Dexcom 同步只落 `glucose_points`（仅留 recordId），不存原始 EGV/event 报文，也不建 `device_sessions`。
- 影响：API 来源数据无完整原始审计轨迹；传感器会话在库内无显式记录（仅靠 device_id 区分）。

### 🟡 LOW

- **L1 — 分析深度落后基线**：缺 MAGE/MODD/CONGA/J-Index 等；MAGE 用固定阈值而非信号峰谷（战略报告点名）。
- **L2 — 无 AGP 可视化**：renderer.py 纯文本，无 10/25/50/75/90 百分位云图。
- **L3 — DECISION_LOG 缺文件**：代码多处引用 `DECISION_LOG D012/D018/D022/D026`，但仓库无 `DECISION_LOG.md`，决策可追溯性受损。
- **L4 — 3 个本地提交未推送**：`97e4391 / c0ad779 / 473c5ae`（Dexcom region+base_url、离线数据集、mock 字段补强）领先 origin/main，未 push。
- **L5 — 知识库内容深度未验证**：`knowledge/authoritative_kb.json` 存在，但战略称"最大技术债在知识库"，其覆盖面/质量需单独评估。
- **L6 — 默认运行库为空**：dev-status `glucose_point_count: 0`（测试均用隔离库），不影响功能。

## 6. 战略层面差异

1. **差异化（整合）已基本成立**：四条线（分析/记忆/RAG/Hermes）确实拼在了一个框架里，这是战略主张的核心，已兑现。
2. **最大技术债仍在知识库**：战略明确指出"技术债不在代码在知识库"。本次审计确认权威 KB 的检索通道（rag 工具）已就绪，但**内容深度**未被验证——这是从"能用"到"可信"的关键一跃，建议作为下一个战略级投入。
3. **临床可读性差距**：AGP 是内分泌科医生看 CGM 的标准方式，当前医生版仅文字。要真正服务"复诊报告"场景，AGP 可视化是硬需求。
4. **分析精度差距**：MAGE 等波动指标缺失会限制"模式发现"的说服力，间接影响协商式假设验证的质量。
